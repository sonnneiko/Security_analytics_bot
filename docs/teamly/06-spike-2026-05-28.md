# Teamly spike — результаты (2026-05-28)

Проверка двух гипотез из дизайна на боевом тенанте `unitpay.teamly.ru`. Тестовая интеграция «Бот СБ», тестовые карточки в таблице «Дашборд взаимодействия СБ/Аккаунтинг» (containerId `49add48b-f23a-4a68-ba26-7115d9ab58a3`).

## Q1: карточка умной таблицы = `article.create`? — НЕТ

Создание карточки шлёт **`tbd.body.create`**, не `article.create`. Сырой payload:

```json
{
  "entityId": "7131093b-d111-4072-b6c5-b52e8a7c208f",
  "entityType": "tbd.body",
  "action": "create",
  "content": { "containerId": "49add48b-f23a-4a68-ba26-7115d9ab58a3" }
}
```

Автора в payload нет. Сразу после `create` прилетает `tbd.body.publish` (тот же entityId) — игнорируем.
Удаление шлёт `tbd.body.garbage` с `entityIds: [...]` (массив) — тоже игнор.

`property.update_value` (при редактировании свойств карточки) **содержит** `content.properties.author` — но это редактирование, не создание, в счёт не идёт.

## Q2: схема чтения автора — `POST /api/v1/wiki/ql/article`

Работает с id строки `tbd.body` (карточка читается как статья):

```
POST {clusterDomain}/api/v1/wiki/ql/article
Headers: Authorization: Bearer <access>, X-Account-Slug: unitpay, Content-Type: application/json
Body:    {"query":{"__filter":{"id":"<entityId>"},"id":true,"title":true,"author":{"id":true,"fullName":true}}}
```

Ответ (плоский объект, без `items`):
```json
{"id":"7131093b-...","title":"тестовая карточка 2","author":{"id":"93cc17b0-ccc2-480b-a05f-38d578751eed","fullName":"Алхазова София"}}
```

- Удалённая карточка → `404 {"error_code":"not_found"}` → трактуем как автор `null` → дроп.
- **`/api/v1/wiki/ql/articles`** (множественное) игнорирует `__filter.id` (отдаёт случайные строки, `{data:[...],paginate:{...}}`). Использовать **только singular** `/article`.

## OAuth

- `POST https://unitpay.teamly.ru/api/v1/auth/integration/authorize` с `client_id + client_secret + redirect_uri + code` → 200 + токены. ✅
- «Ключ авторизации» в UI интеграции = одноразовый `code` (сгорает после первого обмена). При «Обновить» секрета в UI и `code`, и `secret` ротируются вместе.
- `redirect_uri` должен совпадать с тем что в интеграции (у нас `http://localhost`). Несовпадение → 401 `Invalid client`.
- `clusterDomain` из ответа = `https://app.teamly.ru`.

## End-to-end

`tbd.body.create` (id существующей карточки) → дочитка автора → маппинг teamly_user_id→telegram_id → строка в `teamly_events`. Проверено, работает. Реализация: коммит `082004f`.

Транспорт Teamly→бот тестировался через cloudflared quick-tunnel (нестабилен на этой сети — флапает на QUIC, чуть лучше с `--protocol http2`, но всё равно рвётся). Для финальной проверки доставку симулировали POST-ом на localhost; на проде нужен стабильный публичный адрес (Yandex Cloud Serverless Container).

Запросы к TEAMLY AI

Вы можете подключаться к нашему AI-помощнику из сторонних сервисов и получать точные ответы на основе своей базы знаний.

**Как это работает?**


Вы задаёте вопрос в вашем внешнем чате, API собирает информацию из вашей базы знаний в TEAMLY и выводит в вашем внешнем сервисе.

**Как внедрить TEAMLY AI**
**в свой сервис?**

Напишите программу (виджет или бота) для нужного сервиса, а затем подключите его через внешний API к TEAMLY AI.

Вы можете отправлять запросы к TEAMLY AI и получать ответы через **любой внешний сервис** — например, бот Telegram или виджет в Bitrix.





**Запрос к ИИ**

POST /api/v1/ai/chat/message



**Параметры**

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| chatId | uuid|null | Идентификатор чата (необязательный параметр, при использовании поддерживается контекст чата) |
| question\* | string | Вопрос к чату ИИ |
| sourceType\* | string | Тип поиска (Возможные значения: "internet" - поиск в интернете, "documents" - поиск по базе знаний) |

**Пример запроса**

```
{
    "chatId": null,
    "question": "На каком языке написано ядро линукса и что оно из себя представляет",
    "sourceType": "internet"
}
```

**Пример ответа**

```
{
    "token": "97fa66a453725a6de8a8631ae78d5bc4",
    "status": "wait",
    "payload": null
}
```





**HTTP коды ответа**

| Код ответа | Описание |
| --- | --- |
| 200 | Успешно |
| 422 | Некорректное тело запроса |



**_Пояснение к примеру ответа:_**

-   _token - токен данного запроса (необходим для получения ответа)_

-   _status - статус ответа_

-   _payload - содержимое ответа_






**Запрос получения ответа**

GET /api/v1/ai/external/deferred\_result/status?token={token}

_Данный запрос возвращает статус формирования ответа от ИИ и в случае успеха, сам ответ. Формирование ответа от ИИ может занять некоторое время, поэтому запрос не гарантирует возврат ответа от ИИ с первой попытки._

_Параметр token получаем из ответа на предыдущий запрос (POST /api/v1/ai/chat/message). Это идентификатор сообщения к ии, необходимый для получения ответа._

**Пример ответа**

{

"token": "ec5c91d05c93b70d32673fedba92d613",

"status": "done",

"payload": {

"chatId": "179fc895-d7af-4422-b638-6561c2d059b1",

"content": "Фраза \\"виу виу\\" не имеет осмысленного значения в русском языке\\\\. Возможно, вы имели в виду что\\\\-то другое? Если у вас есть конкретный вопрос или тема для обсуждения, я с удовольствием помогу вам\\\\."

}

}



**Параметры**

| Параметр | Описание |
| --- | --- |
| token | Токен данного запроса (необходим для получения ответа) |
| status | Статус ответа |
| payload | Содержимое ответа |
| chatId | Идентификатор чата (используется для поддержания контекста) |
| content | Ответ от ИИ |



**Типы статуса ответа**

| Статус | Значение |
| --- | --- |
| done | Ответ получен |
| empty | Ответ по токену очищен, информация больше получить нельзя |
| error | Произошла ошибка |
| wait | Ответ в процессе формирования |

_Информация об ответе живёт 5 минут._
